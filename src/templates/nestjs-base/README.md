# becraft NestJS Starter

> Production-grade backend starter — Contract-Driven Development

## 🚀 Quick Start

```bash
# 1. Install
npm install

# 2. Set environment
cp .env.example .env
# Generate secrets:
echo "JWT_SECRET=$(openssl rand -base64 32)" >> .env
echo "JWT_REFRESH_SECRET=$(openssl rand -base64 32)" >> .env

# 3. Start dependencies (PostgreSQL + Redis)
docker-compose up -d postgres redis

# 4. Run migrations (after adding entities)
npx prisma migrate dev

# 5. Start dev server
npm run start:dev
```

Open:
- 📖 OpenAPI docs: http://localhost:3000/docs
- 🏥 Health: http://localhost:3000/health
- 📊 Liveness: http://localhost:3000/health/live
- ✅ Readiness: http://localhost:3000/health/ready

## 📦 Built-In

- ✅ **NestJS 10** with TypeScript strict mode
- ✅ **Prisma 5** for PostgreSQL
- ✅ **Pino** structured logging with PII redaction
- ✅ **OpenAPI** auto-generated at /docs
- ✅ **Validation** with class-validator + Zod env validation
- ✅ **Health checks** (liveness + readiness)
- ✅ **Rate limiting** via @nestjs/throttler
- ✅ **Helmet** security headers
- ✅ **CORS** configurable
- ✅ **API versioning** (URI-based: /api/v1/...)
- ✅ **RFC 7807** error responses
- ✅ **Request-id** propagation
- ✅ **Graceful shutdown**
- ✅ **Docker** multi-stage build

## 🤖 Use becraft Agents

Generate code with agents:

```
/be-schema add User entity
/be-api create CRUD for users
/be-auth setup JWT
/be-test generate tests
/be-observe add metrics
```

Or build everything at once:

```
/be-bootstrap user management API with JWT auth
```

## 📁 Project Structure

```
.
├── prisma/
│   ├── schema.prisma     # DB schema (add models here)
│   └── migrations/       # Auto-generated by Prisma
├── src/
│   ├── main.ts           # App bootstrap
│   ├── app.module.ts     # Root module
│   ├── common/           # Shared utilities
│   │   ├── decorators/   # @Public, @Roles, @CurrentUser
│   │   ├── dto/          # PaginationDto, etc.
│   │   ├── filters/      # AllExceptionsFilter (RFC 7807)
│   │   ├── guards/       # (added by /be-auth)
│   │   ├── interceptors/ # Logging, metrics
│   │   ├── middleware/   # request-id
│   │   └── pipes/
│   └── modules/
│       ├── prisma/       # PrismaService
│       ├── health/       # /health endpoints
│       └── (your modules)
├── test/                 # E2E tests
├── docker-compose.yml
├── Dockerfile
└── package.json
```
