# becraft Agents

6 specialized agents that power the `/be-*` commands. Each is **self-sufficient**, **self-correcting**, and **expert-level**.

## 🏗️ Dual Architecture

becraft supports **two agent formats** for different IDEs:

```
src/agents/
├── *.md              ← Antigravity-friendly format
└── subagents/
    └── *.md          ← Claude Code native format
```

### Format Comparison

| Aspect | Antigravity (root) | Claude Code (subagents/) |
|--------|--------------------|--------------------------|
| **For IDE** | Google Antigravity | Claude Code |
| **Frontmatter** | `type`, `skills`, `triggers` | `tools`, `model` |
| **Invocation** | Inlined into workflow files | Native `Task` tool delegation |
| **Body content** | Same | Same |
| **Generated by** | `scripts/bundle.js` | Manual (then synced via script) |

### Antigravity Format
```yaml
---
name: schema-architect
type: sub-agent
description: >
  Expert PostgreSQL + Prisma schema architect.
skills:
  - schema-design
  - contract-first
triggers:
  - /be-schema command
---
```

### Claude Code Format
```yaml
---
name: schema-architect
description: |
  Expert PostgreSQL + Prisma schema architect.
  Delegate when: schema design, migrations, indexes.
tools: [Read, Write, Edit, Bash]
model: sonnet
---
```

## 📦 Installation Paths

When user runs `becraft install`:

| IDE | Agent Location | Format Used |
|-----|----------------|-------------|
| **Claude Code** | `.claude/agents/*.md` | Claude native (from `subagents/`) |
| **Antigravity** | Embedded in `.agent/workflows/be-*.md` | Antigravity (bundled from root) |


Plus `.be/agents/` for reference (Antigravity format).

## 🧠 Agent Philosophy

Every agent follows three core tenets:

### 1. Self-Sufficient
- No dependency on other agents during work
- No need for additional input
- No unnecessary clarifying questions

### 2. Self-Correcting
- Verify own work before declaring done
- Auto-fix errors silently (max 5 attempts)
- Don't wait for user to point out issues

### 3. Expert-Level
- Make correct decisions
- Apply best practices automatically
- No amateur mistakes

## 🛠️ Universal Sections

Every agent file (~300-650 lines) contains:

1. **YAML frontmatter** (format depends on target)
2. **🚨 Memory Protocol** — read 9 / write relevant files
3. **📢 Agent Announcement** — Start/Parallel/Complete templates
4. **Identity** — name, role, expertise, motto
5. **🧠 Ultrathink Principles** — 4 points
6. **⚡ Parallel Execution** — CAN / MUST WAIT declarations
7. **`<default_to_action>`** — when to act vs ask
8. **`<use_parallel_tool_calls>`** — efficiency
9. **`<investigate_before_answering>`** — what to read first
10. **🛠️ Skills Integration** — list of skills used
11. **🔄 Workflow** — 5 phases (Memory → Investigate → Design → Build → Verify → Report)
12. **Code Patterns** — actual templates
13. **Error Recovery** — common errors + fixes
14. **Quality Standards** — Must Have / Must NOT Have
15. **Self-Improvement Protocol** — questions to ask after work
16. **📝 Response Format** — 3-section template

## 🔄 Sync Strategy

When updating an agent:

1. Edit the **Claude Code native** version at `subagents/<name>.md`
2. Run regenerate script to sync Antigravity format:
   ```bash
   bash scripts/sync-antigravity-agents.sh
   ```
3. Run bundler to regenerate workflows:
   ```bash
   npx becraft bundle
   ```
4. Commit all changes

---

## 🎯 Adding a New Agent

1. Create `subagents/new-agent.md` (Claude Code format)
2. Create `new-agent.md` at root (Antigravity format) — same body, different frontmatter
3. Add command file `src/commands/be-new.md` that delegates to it
4. Update `BUNDLE_MAP` in `scripts/bundle.js`
5. Update `installer/ide-handlers/claude-code.js` to register the agent
6. Update CLAUDE.md template to mention the new command
7. Run `node scripts/bundle.js` to regenerate
8. Update this README + main README
